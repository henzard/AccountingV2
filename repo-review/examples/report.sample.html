Executive summary

A white-box review of both halves of NutSync — the offline-first Expo/Electron client and its Node-RED sync hub — covering security (red & blue team), replication correctness, API design, database, backend code quality, UX/accessibility, design-system consistency, and test quality. Findings were documented as audit reports, filed as 116 tracked issues, and the Critical/High security defects were fixed and verified. Finally, the team's enterprise engineering discipline (six-layer test matrix, anti-pattern catalogue, branch hygiene, CI, review skills) was adapted into both repos.

Honest caveat: an LLM-assisted review plus scanners substantially reduces risk but cannot prove a system is unhackable or bug-free. Treat this as one strong layer; keep a human reviewer on anything touching auth, crypto, and the sync conflict resolution.

Replication correctness (offline-first)

Audited as a distributed system. Convergence does not hold in general; conflict resolution is wall-clock last-write-wins at one-second resolution with direction-dependent ties.

PropertyHolds?Note
Atomicity (per-merge transaction)YesBEGIN/COMMIT/ROLLBACK both sides — the one strong property
ConvergenceNobroken by silent skips + clock-skew LWW
No lost committed writeNoC1 (un-retried skip) + C2 (markSynced race) — both Critical
Idempotent applyEdge yes / hub nohub re-stamps updatedAt on apply
Delete safetyPartialtombstones propagate, but delete-update can resurrect

Recommended: monotonic row_version + hub sequence, only-if-unchanged markSynced, a durable dead-letter table, and a fault-injection property-test harness.

Scoring methodology (reusable across systems)

Apply the same rubric to every system you review for an apples-to-apples comparison. Each dimension is scored 0–100 as-built, then weighted. For security and data integrity the system inherits its weakest component (a system is only as safe as its weakest half) rather than an average.

DimensionWeightWhat it measures
Security & secrets25%Auth, injection, secrets handling, transport, access control — weighted highest because it's where vibe-coded systems fail hardest.
Data integrity & sync20%Can a committed write be silently lost/corrupted? Convergence, idempotency, conflict resolution, transactions.
Architecture & code quality15%SOLID, layering, duplication, complexity, deployed==tested.
Data layer10%Schema/types/constraints, indexing, migrations, query quality.
API & contracts10%REST correctness, status codes, validation, versioning, idempotency, docs.
Testing & verifiability10%Real coverage + test authenticity (not mock-echo theatre); CI gates.
UX & accessibility5%Heuristics, WCAG, error/loading states, flows.
UI / code consistency5%Design tokens vs magic values, reuse vs duplication, naming.
